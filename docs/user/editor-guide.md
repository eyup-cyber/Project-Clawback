## Editor Responsibilities

As an Editor, you are responsible for:

### Primary Duties

- **Quality Control**: Ensure content meets platform standards
- **Content Review**: Approve or request changes on submissions
- **Editorial Support**: Help contributors improve their work
- **Community Standards**: Enforce community guidelines
- **Content Curation**: Feature and promote quality content

### Secondary Duties

- Provide feedback to contributors
- Track content performance
- Report issues to administrators
- Mentor new contributors

## Review Queue

### Queue Overview

The review queue shows all content awaiting editorial review:

- **Title**: Content title
- **Author**: Who submitted it
- **Type**: Article, video, audio, or visual
- **Submitted**: When it was submitted
- **Status**: Review stage

### Queue Filters

Filter content by:

- Content type
- Submission date
- Author
- Priority level
- Previous review status

### Priority Levels

1. **High**: Featured content requests, time-sensitive
2. **Normal**: Standard submissions
3. **Low**: Updates to existing content

## Content Review Process

### Step 1: Initial Assessment

When reviewing content:

1. Open the submission from the queue
2. Read/view the content completely
3. Check basic requirements:
   - [ ] Title is appropriate
   - [ ] Content is complete
   - [ ] Category is correct
   - [ ] Media files load properly

### Step 2: Quality Check

Verify content quality:

- [ ] Original content (not plagiarized)
- [ ] Accurate information
- [ ] Proper formatting
- [ ] Good grammar and spelling
- [ ] Appropriate images/media

### Step 3: Guidelines Compliance

Ensure compliance with:

- [ ] Community guidelines
- [ ] Content policies
- [ ] Age-appropriate content
- [ ] Copyright requirements
- [ ] Disclosure requirements (if applicable)

### Step 4: SEO Review

Check SEO elements:

- [ ] Title (50-60 characters)
- [ ] Meta description (150-160 characters)
- [ ] Keywords used naturally
- [ ] Alt text for images
- [ ] Internal links where appropriate

### Step 5: Decision

Make your editorial decision:

- **Approve**: Ready to publish
- **Request Changes**: Needs revisions
- **Reject**: Does not meet standards

## Editorial Standards

### Content Quality

**Excellent content:**

- Provides unique value
- Well-researched and accurate
- Engaging and well-written
- Properly formatted
- Includes quality media

**Common issues to address:**

- Thin or shallow content
- Poor formatting
- Grammatical errors
- Missing citations
- Low-quality images

### Formatting Guidelines

#### Headings

- Use H2 for main sections
- Use H3 for subsections
- Keep hierarchy logical

#### Paragraphs

- Keep paragraphs focused
- 2-4 sentences ideal
- Use white space effectively

#### Lists

- Use for scannable content
- Keep items parallel
- Don't over-use

#### Media

- High-quality images (min 800px wide)
- Relevant captions
- Proper alt text
- Optimized file sizes

## Making Edits

### Editor Permissions

Editors can:

- Suggest edits
- Make direct corrections (typos, formatting)
- Add editorial notes
- Update metadata
- Request changes from author

### Direct Edits

For minor issues, make direct corrections:

1. Click **Edit** on the content
2. Make necessary changes
3. Add a note explaining changes
4. Save as editorial revision

### Suggested Edits

For larger changes:

1. Click **Suggest Edit**
2. Highlight the section
3. Write your suggestion
4. Send to author for approval

### Feedback Best Practices

- Be specific and constructive
- Explain the "why" behind changes
- Provide examples when helpful
- Encourage improvement
- Be timely with feedback

## Publishing Decisions

### Approval Workflow

```
Submitted ‚Üí In Review ‚Üí Changes Requested ‚Üí Revised ‚Üí Approved ‚Üí Scheduled ‚Üí Published
```

### Approval

When approving content:

1. Complete all review steps
2. Click **Approve for Publishing**
3. Choose publishing option:
   - Publish immediately
   - Schedule for later
   - Save as approved (author publishes)

### Requesting Changes

When requesting changes:

1. Document specific issues
2. Provide clear guidance
3. Set deadline if urgent
4. Click **Request Changes**
5. Send to author

Example feedback:

```
Hi [Author],

Great article on [topic]! A few things to address before publishing:

1. Heading structure - Your H2s and H3s need reorganizing for clarity
2. Second section - Please add a source for the statistics mentioned
3. Images - The third image needs alt text

Please resubmit when you've made these changes. Let me know if you have questions!
```

### Rejection

When rejecting content:

1. Document clear reasons
2. Be respectful in explanation
3. Offer constructive guidance
4. Click **Reject**
5. Send detailed feedback

Rejection reasons:

- Does not fit platform
- Quality issues (with specifics)
- Policy violations
- Duplicate content
- Inappropriate content

## Content Moderation

### Handling Reports

When content is reported:

1. Review the report details
2. Examine the content in question
3. Check against policies
4. Make determination:
   - No action needed
   - Warning to author
   - Edit required
   - Content removal
   - Escalate to admin

### Report Categories

| Category       | Response Time |
| -------------- | ------------- |
| Harassment     | 4 hours       |
| Misinformation | 8 hours       |
| Spam           | 24 hours      |
| Copyright      | 48 hours      |
| Other          | 72 hours      |

### Taking Action

**Warning**: Minor or first-time violations

- Document the issue
- Send clear warning
- Note in contributor record

**Edit Required**: Fixable violations

- Specify required changes
- Set deadline
- Unpublish until fixed

**Removal**: Serious violations

- Remove content immediately
- Document thoroughly
- Notify author with explanation

**Escalation**: Severe or repeated issues

- Flag for admin review
- Do not take unilateral action
- Preserve evidence
